# Contributing

Thanks for helping improve Beacon. This project emphasizes governance, explicit consent, and observability.

## How to propose changes

- Open an issue to discuss scope and intent.
- Keep changes minimal and auditable.
- For larger work, outline an approach before implementation.

## Pull request guidelines

- Link the relevant issue.
- Explain security and privacy impact.
- Prefer small, focused PRs.
- Include tests or a clear reason why tests are not applicable.

## Code style

- Prefer clear, conservative abstractions.
- Avoid unnecessary dependencies.
- Keep docs short, structured, and factual.

## Development

- Run tests and lint checks when available.
- Document any new approvals, capabilities, or logging behavior.
